# Black Project Sip — каталог проектов домов

Статический сайт-каталог. Без фреймворков и сборщиков: один HTML-файл + данные в
`catalog.data.js` + папки с картинками. Всё содержимое управляется **папками
проектов** и одним скриптом `build.py`.

---

## ⭐ Главное правило

> **Ты меняешь только папки в `projects/`. Потом запускаешь `python3 build.py`.**
> Скрипт сам найдёт картинки, сделает миниатюры и пересоберёт каталог.

`catalog.data.js` **руками не редактируется** — он генерируется. Любая ручная
правка пропадёт при следующей сборке.

---

## Структура

```
Сайт/
├─ index.html          ← сам сайт (открывается в браузере)
├─ catalog.data.js     ← АВТО-генерируется build.py (не трогать)
├─ build.py            ← сборщик: картинки → миниатюры + каталог
├─ README.md           ← этот файл
└─ projects/           ← ВСЕ проекты, по папке на проект
   ├─ 01/
   │  ├─ project.json  ← метаданные (площадь, габариты, этажи…)
   │  ├─ facade.png    ← фасад
   │  ├─ plan.png      ← планировка
   │  ├─ 3d-1.jpg      ← 3D-вид (фронт)        — если есть
   │  ├─ 3d-2.jpg      ← 3D-вид (ракурс)       — если есть
   │  └─ thumbs/       ← миниатюры (генерит build.py, не трогать)
   ├─ 02/
   └─ …
```

Номер папки = номер проекта на сайте (`01`, `02`, … сплошная нумерация).

## Соглашение по именам картинок

`build.py` распознаёт файлы по началу имени (регистр и формат не важны —
`.png/.jpg/.jpeg/.webp`):

| Начинается с | Что это      | Сколько          |
|--------------|--------------|------------------|
| `facade`     | фасад        | обычно 1         |
| `plan`       | планировка   | 1 или несколько  |
| `3d`         | 3D-вид       | 0, 1 или 2       |

Если файлов одного типа несколько — нумеруй: `plan-1.png`, `plan-2.png`,
`3d-1.jpg`, `3d-2.jpg`. Порядок — натуральный (`plan-2` раньше `plan-10`).

**Порядок показа в просмотрщике:** сначала 3D-виды, затем фасад, затем планировки.
На обложке карточки по умолчанию — 3D (если есть), иначе фасад.

## `project.json` — метаданные проекта

```json
{
  "area": 142.5,      // площадь застройки, м²   (число)
  "width": 15,        // ширина, м               (число)
  "depth": 9.5,       // глубина, м              (число)
  "floors": 1,        // этажность: 1 или 2
  "pages": "1-2",     // стр. исходного PDF (для поиска); можно ""
  "note": ""          // заметка (напр. "баня", "терраса")
}
```

Любое поле можно оставить пустым/убрать — сайт это переживёт. Если не указать
`floors`, скрипт поставит 2 при двух планировках, иначе 1.

## Рецепты

> После **любого** изменения запусти: `python3 build.py`

### ➕ Добавить новый проект
1. Создай папку `projects/83/` (следующий свободный номер).
2. Положи картинки: `facade.png`, `plan.png` (и `3d-1.jpg`/`3d-2.jpg`, если есть).
3. Создай `projects/83/project.json` (см. шаблон выше).
4. `python3 build.py`

### ➖ Удалить проект
Удали папку `projects/NN/` целиком → `python3 build.py`.

### 🔄 Заменить картинку
Положи новый файл под тем же именем (перезапиши) → `python3 build.py`.
Миниатюра пересоздастся автоматически. Картинку можно кидать любого размера —
скрипт сам ужмёт большие до 1600px.

### 🧩 Добавить ещё одну планировку
Переименуй существующую в `plan-1.png` и добавь `plan-2.png` (и т.д.) →
`python3 build.py`. Этажность станет «2», если планировок ≥2 и в `project.json`
не задано иное.

### 🏠 Добавить 3D-виды проекту (напр. где их пока нет — 01, 29, 63)
Положи в папку проекта `3d-1.jpg` и (по желанию) `3d-2.jpg` → `python3 build.py`.
Оригиналы рендеров лежат в `../_source/3d-renders/` (имена — по номеру дома).
Сопоставить дом ↔ проект помогает доска `../_tools/match-3d.html`.

### ✏️ Поправить площадь / этажность / заметку
Отредактируй `projects/NN/project.json` → `python3 build.py`.

## Запуск сборщика

Нужен **Python 3** и библиотека **Pillow** (один раз):

```bash
pip3 install Pillow
```

Затем из папки `Сайт/`:

```bash
python3 build.py          # обычная сборка
python3 build.py --force  # пересоздать ВСЕ миниатюры с нуля
```

Скрипт печатает отчёт и предупреждения (нет фасада, битый `project.json` и т.п.).

---

## Посмотреть сайт локально

Просто открыть `index.html` файлом не получится (браузер не отдаёт соседние
файлы). Подними локальный сервер из папки `Сайт/`:

```bash
python3 -m http.server 8000
```

и открой <http://localhost:8000/index.html>.

---

## Деплой

Сайт статический — заливается как есть (GitHub Pages, любой хостинг статики).
Перед заливкой выполни `python3 build.py`, чтобы `catalog.data.js` и миниатюры
были актуальны. Папки `../_source/` и `../_tools/` к сайту **не относятся** —
их выкладывать не нужно.

---

## Что где (вне папки сайта)

- `../_source/` — исходники, сайтом не используются: оригиналы рендеров (PNG
  5504px), исходные страницы PDF, `_площади.md`, сам PDF.
- `../_tools/match-3d.html` — доска для сопоставления 3D-видов с проектами.
